# DigEino UI/UX 工具优化方案

> 基于 impeccable 项目的分析，为 DigEino 的 UI/UX 工具添加命令式审查工具、增强反模式检测、扩展领域参考文档，并改进 AI Slop 检测机制

## 一、项目对比分析

### Impeccable 项目核心优势

1. **命令式工具集**：17 个细粒度命令（/audit, /critique, /normalize, /polish 等），提供操作性指导
2. **反模式识别系统**：系统化的"AI Slop"检测，明确告诉 AI 不应该做什么
3. **领域参考文件**：7 个详细参考文档（Typography, Color, Spatial, Motion, Interaction, Responsive, UX Writing）
4. **审查和优化工具**：技术质量审查（/audit）和 UX 设计审查（/critique）
5. **明确的"不要做什么"指导**：避免通用 AI 美学（Inter 字体、紫色渐变、卡片嵌套等）

### DigEino 当前优势

1. **知识库检索**：BM25 搜索算法，400+ 设计规则和模式
2. **设计系统自动生成**：完整的生成流程（产品类型匹配 → 多领域搜索 → 推理引擎 → 输出）
3. **推理引擎**：100 条行业特定规则（SaaS、E-commerce、Healthcare 等）
4. **持久化存储**：Master + Overrides 模式，支持应用隔离
5. **技术栈支持**：React, Vue, Tailwind, SwiftUI, Flutter 等

### 差距分析

| 维度         | Impeccable        | DigEino        | 差距               |
| ------------ | ----------------- | -------------- | ------------------ |
| 工具类型     | 命令式、操作性    | 生成式、建议性 | 缺少审查和优化工具 |
| 反模式检测   | 系统化、明确      | 有但不够系统   | 需要增强           |
| 参考文档     | 7 个详细领域文档  | CSV 数据文件   | 缺少结构化参考文档 |
| AI Slop 检测 | 专门的检测机制    | 无             | 需要添加           |
| 审查能力     | /audit, /critique | 无             | 需要添加           |

## 二、优化方案

### 阶段一：增强反模式检测系统

#### 1.1 扩展反模式数据库（CSV 格式）

**目标**：在 CSV 文件中增加更多字段列，使反模式信息更完整和结构化

**数据存储格式**：保持 CSV 格式，与现有 `ui-reasoning.csv`、`styles.csv` 等数据文件保持一致

**实现**：

- 创建 `tools/ui_ux/data/anti_patterns.csv`，使用 CSV 格式存储，包含以下列：
  - `Name`：反模式名称
  - `Description`：描述
  - `Severity`：严重程度（Critical/High/Medium/Low）
  - `Detection`：检测规则（正则表达式或关键词）
  - `Recommendation`：修复建议
  - `Command`：相关命令（audit/critique/normalize）
  - `Category`：分类（Typography/Color/Layout/Motion/Interaction 等）
  - `Examples`：示例代码或场景（可选）

**示例数据**（CSV 格式）：

```csv
Name,Description,Severity,Detection,Recommendation,Command,Category,Examples
"Overused Fonts","使用过度使用的字体（Inter, Arial, Roboto）",High,"font-family.*(Inter|Arial|Roboto|Open Sans)","使用更独特的字体，如 Instrument Sans, Plus Jakarta Sans",audit,Typography,"font-family: Inter, sans-serif"
"Gray on Color","在彩色背景上使用灰色文字",Critical,"background.*#[0-9a-f]{3,6}.*color.*gray","使用背景色的深色变体",critique,Color,"background: #3b82f6; color: #6b7280"
"Nested Cards","卡片嵌套卡片",Medium,"\.card.*\.card","扁平化层次结构，移除不必要的容器",normalize,Layout,"<div class='card'><div class='card'>...</div></div>"
```

**优势**：

- CSV 格式易于编辑和维护（Excel、文本编辑器）
- 与现有数据文件格式一致
- 支持版本控制和协作编辑
- 便于后续扩展新字段列

#### 1.2 反模式检测引擎

**文件**：`tools/ui_ux/anti_pattern_detector.go`

**功能**：

- 读取 `anti_patterns.csv` 文件（使用 `//go:embed` 嵌入，与现有方式一致）
- 解析 CSV 数据到内存结构体（运行时处理，不改变存储格式）
- 扫描代码/设计系统，使用检测规则匹配反模式
- 生成反模式报告（位置、严重程度、修复建议）
- 集成到审查工具中

**实现方式**：

- 使用 `encoding/csv` 包读取 CSV
- 与现有 `service.go` 中读取 CSV 的方式保持一致
- 检测引擎只是读取和处理 CSV，不改变数据存储格式

### 阶段二：添加命令式审查工具

**使用方式说明**：

这些审查工具是作为 **Eino 工具（tool.BaseTool）** 实现的，**仅通过 LLM Agent 使用**：

1. **通过 LLM Agent 使用**：
   - 用户通过自然语言告诉 Agent："审查这个组件的代码质量" 或 "检查这个页面的 UX 设计"
   - Agent 会根据工具描述自动选择并调用相应的工具（`ui_ux_audit` 或 `ui_ux_critique`）
   - Agent 会解析工具返回的结果，并以用户友好的方式呈现

2. **工具注册**：
   - 所有工具通过 `tools.BaseTools()` 函数注册
   - 集成到 Agent 系统后，Agent 可以自动发现和使用这些工具

**与 Impeccable 的区别**：
- Impeccable：命令式（如 `/audit`, `/critique`），用户直接输入命令
- DigEino：工具式，通过 Agent 调用，用户用自然语言与 Agent 交互，无需记忆具体命令

#### 2.1 技术质量审查工具（ui_ux_audit）

**文件**：`tools/ui_ux/audit_tool.go`

**工具名称**：`ui_ux_audit`

**工具描述**：对前端代码进行全面的技术质量审查，包括可访问性、性能、主题、响应式设计和反模式检测。生成详细的审查报告，包含问题位置、严重程度和修复建议。

**功能**：

- 可访问性检查（对比度、ARIA、键盘导航）
- 性能检查（动画性能、布局抖动、资源优化）
- 主题检查（硬编码颜色、深色模式支持）
- 响应式设计检查（固定宽度、触摸目标、水平滚动）
- 反模式检查（集成反模式检测引擎）

**参数**：

- `path`：要审查的文件或目录路径（必填）
- `area`：要审查的特定区域/功能（可选，如 "header", "form"）
- `severity_filter`：严重程度过滤（可选，如 "critical,high"）

**输出格式**：

```json
{
  "anti_patterns_verdict": "FAIL - 检测到 5 个 AI Slop 特征",
  "executive_summary": {
    "total_issues": 12,
    "critical": 2,
    "high": 4,
    "medium": 4,
    "low": 2
  },
  "findings": [
    {
      "location": "components/Header.tsx:45",
      "severity": "Critical",
      "category": "Anti-Pattern",
      "description": "使用 Inter 字体（过度使用的字体）",
      "impact": "设计缺乏独特性，看起来像 AI 生成",
      "recommendation": "使用 Instrument Sans 或 Plus Jakarta Sans",
      "suggested_command": "normalize"
    }
  ]
}
```

#### 2.2 UX 设计审查工具（ui_ux_critique）

**文件**：`tools/ui_ux/critique_tool.go`

**工具名称**：`ui_ux_critique`

**工具描述**：从 UX 设计角度评估界面效果，包括 AI Slop 检测、视觉层次、信息架构、情感共鸣等。提供设计改进建议和优先级排序。

**功能**：

- AI Slop 检测（最重要的检查）
- 视觉层次评估
- 信息架构评估
- 情感共鸣评估
- 可发现性和可操作性评估
- 排版评估
- 颜色使用评估
- 状态和边界情况评估

**参数**：

- `path`：要审查的文件或目录路径（必填）
- `area`：要审查的特定区域/功能（可选）
- `focus`：重点关注领域（可选，如 "visual-hierarchy", "color-usage"）

**输出格式**：

```json
{
  "anti_patterns_verdict": "FAIL - 看起来像 AI 生成的界面",
  "overall_impression": "设计缺乏独特性，使用了常见的 AI 模板",
  "whats_working": ["良好的响应式布局", "清晰的导航结构"],
  "priority_issues": [
    {
      "what": "使用 AI 颜色调色板（青色+深色背景）",
      "why_it_matters": "缺乏品牌个性，看起来像模板",
      "fix": "使用与品牌相关的配色方案",
      "command": "colorize"
    }
  ]
}
```

#### 2.3 设计系统标准化工具（ui_ux_normalize）

**文件**：`tools/ui_ux/normalize_tool.go`

**工具名称**：`ui_ux_normalize`

**工具描述**：分析代码与设计系统的偏差，生成标准化计划并执行标准化。确保代码使用设计令牌、统一组件和模式。

**功能**：

- 发现设计系统文档（自动搜索 MASTER.md 或设计系统配置文件）
- 分析当前实现与设计系统的偏差
- 生成标准化计划
- 执行标准化（使用设计令牌、替换组件、统一模式）

**参数**：

- `path`：要标准化的文件或目录路径（必填）
- `feature`：要标准化的特定功能/页面（可选）
- `design_system_path`：设计系统文档路径（可选，自动发现）
- `dry_run`：仅生成计划，不执行修改（可选，默认 false）

### 阶段三：扩展领域参考文档

#### 3.1 参考文档存储方案选择

**分析**：

参考文档包含两种类型的内容：

1. **结构化数据**：原则、规则、最佳实践（适合 CSV）
2. **长文本内容**：详细说明、代码示例、层次化章节（适合 Markdown）

**推荐方案：CSV 为主，Markdown 为辅**

**理由**：

- ✅ **与现有系统一致**：DigEino 已使用 CSV 存储结构化数据（styles.csv, ux-guidelines.csv）
- ✅ **易于编辑和维护**：CSV 可以用 Excel、文本编辑器编辑，便于协作
- ✅ **便于查询和检索**：CSV 数据可以直接用 BM25 搜索，与现有搜索系统集成
- ✅ **版本控制友好**：CSV 文件易于 diff 和合并
- ✅ **代码示例处理**：参考 `ux-guidelines.csv` 的做法，代码示例可以作为 CSV 的一个字段

**目录结构**（CSV 方案）：

```
tools/ui_ux/data/
├── reference/
│   ├── typography.csv          # 排版参考（CSV 格式）
│   ├── color_and_contrast.csv  # 颜色与对比度（CSV 格式）
│   ├── spatial_design.csv     # 空间设计（CSV 格式）
│   ├── motion_design.csv      # 动效设计（CSV 格式）
│   ├── interaction_design.csv # 交互设计（CSV 格式）
│   ├── responsive_design.csv  # 响应式设计（CSV 格式）
│   └── ux_writing.csv         # UX 文案（CSV 格式）
```

#### 3.2 CSV 参考文档结构设计

**参考 `ux-guidelines.csv` 的结构**，每个参考文档 CSV 包含以下字段：

**基础字段**：

- `Category`：分类/主题（如 "Vertical Rhythm", "Modular Scale"）
- `Topic`：具体主题名称
- `Description`：详细描述（可以包含多段文本）
- `Do`：最佳实践（DO）
- `Don't`：反模式（DON'T）
- `Code Example Good`：好的代码示例
- `Code Example Bad`：坏的代码示例
- `Severity`：严重程度（Critical/High/Medium/Low）
- `Related Patterns`：相关模式（逗号分隔）

**扩展字段**（可选）：

- `Principles`：核心原则
- `Tools`：推荐工具和资源
- `References`：参考链接
- `Keywords`：关键词（用于搜索）

**示例：`typography.csv`**

```csv
Category,Topic,Description,Do,Don't,Code Example Good,Code Example Bad,Severity,Keywords
"Classic Principles","Vertical Rhythm","Your line-height should be the base unit for ALL vertical spacing. If body text has line-height: 1.5 on 16px type (= 24px), spacing values should be multiples of 24px.","Use line-height as base unit for spacing","Use arbitrary spacing values","body { line-height: 1.5; } .section { margin-bottom: 48px; } /* 2x line-height */","body { line-height: 1.5; } .section { margin-bottom: 50px; }",High,"vertical rhythm,line-height,spacing"
"Classic Principles","Modular Scale","Use fewer sizes with more contrast. A 5-size system covers most needs.","Use modular scale (1.25, 1.333, 1.5)","Use too many similar sizes","font-size: clamp(1rem, 2vw + 0.5rem, 2rem);","font-size: 14px; font-size: 15px; font-size: 16px;",High,"modular scale,type scale,hierarchy"
"Font Selection","Choosing Distinctive Fonts","Avoid the invisible defaults: Inter, Roboto, Open Sans.","Use distinctive fonts like Instrument Sans, Plus Jakarta Sans","Use overused fonts (Inter, Roboto, Arial)","font-family: 'Instrument Sans', sans-serif;","font-family: Inter, sans-serif;",High,"font selection,distinctive fonts,Inter alternative"
"Web Font Loading","Font Display Strategy","The layout shift problem: fonts load late, text reflows.","Use font-display: swap and match fallback metrics","Ignore font loading performance","@font-face { font-display: swap; size-adjust: 105%; }","@font-face { font-family: CustomFont; src: url(font.woff2); }",Medium,"font loading,FOUT,layout shift"
```

**优势**：

- ✅ 结构化数据，便于查询和过滤
- ✅ 代码示例作为字段，便于程序处理
- ✅ 与现有 CSV 数据格式一致
- ✅ 支持 BM25 搜索（在 Description、Do、Don't 字段中搜索）

**注意事项**：

- CSV 字段中的换行符需要用引号包裹
- 长文本可以存储在 Description 字段中
- 代码示例存储在专门的字段中，便于格式化显示

#### 3.3 参考文档检索工具

**文件**：`tools/ui_ux/reference_tool.go`

**功能**：

- 读取参考文档 CSV 文件（使用 `//go:embed` 嵌入）
- 根据领域/关键词检索相关参考文档
- 在审查工具中自动引用相关参考文档
- 提供上下文相关的设计指导

**集成方式**：

- 在 `CSVConfigs` 中添加参考文档配置：

```go
"reference_typography": {
    File:       "data/reference/typography.csv",
    SearchCols: []string{"Category", "Topic", "Description", "Do", "Don't", "Keywords"},
    OutputCols: []string{"Category", "Topic", "Description", "Do", "Don't", "Code Example Good", "Code Example Bad", "Severity"},
},
```

- 在 `service.go` 中添加 `SearchReference(domain, query, maxResults)` 方法
- 审查工具（audit、critique）可以调用参考文档检索，提供上下文相关的设计指导

#### 3.4 可选：Markdown 补充文档

如果需要更详细的说明文档（如完整的设计系统指南），可以创建 Markdown 文件作为补充：

```
tools/ui_ux/reference/
├── typography.md          # 详细排版指南（可选）
├── color_and_contrast.md  # 详细颜色指南（可选）
└── ...
```

**使用场景**：

- 当 CSV 字段无法容纳超长文本时
- 需要层次化章节结构时
- 需要大量代码示例和图表时

**实现方式**：

- 使用 `//go:embed reference/*.md` 嵌入
- 提供 `GetReferenceMarkdown(domain)` 方法
- 在工具输出中提供 Markdown 链接或内容片段

### 阶段四：AI Slop 检测机制

#### 4.1 AI Slop 特征库（CSV 格式）

**文件**：`tools/ui_ux/data/ai_slop_patterns.csv`

**数据存储格式**：保持 CSV 格式，与现有数据文件保持一致

**包含字段列**：

- `Pattern`：AI Slop 特征名称
- `Description`：描述
- `Weight`：权重（用于计算 AI Slop 分数）
- `Detection`：检测规则（正则表达式或关键词）
- `Category`：分类（Color/Typography/Layout/Effect/Animation）
- `Examples`：示例代码或场景

**示例数据**（CSV 格式）：

```csv
Pattern,Description,Weight,Detection,Category,Examples
"AI Color Palette","AI 颜色调色板（青色+深色、紫色渐变）",10,"(cyan|#00ffff|#06b6d4).*dark|purple.*gradient",Color,"background: #0a0e27; color: #00ffff"
"Overused Fonts","过度使用的字体（Inter, Arial, Roboto）",8,"font-family.*(Inter|Arial|Roboto)",Typography,"font-family: Inter, sans-serif"
"Hero Metric Layout","Hero 指标布局模板",7,"big.*number.*small.*label.*stats",Layout,"<div class='hero-metric'>...</div>"
"Glassmorphism","玻璃态效果",6,"backdrop-filter.*blur|glass.*card",Effect,"backdrop-filter: blur(10px)"
"Bounce Easing","弹跳缓动",5,"ease.*bounce|elastic.*easing",Animation,"transition: all 0.3s cubic-bezier(0.68, -0.55, 0.265, 1.55)"
```

#### 4.2 AI Slop 检测器

**文件**：`tools/ui_ux/ai_slop_detector.go`

**功能**：

- 读取 `ai_slop_patterns.csv` 文件（使用 `//go:embed` 嵌入）
- 解析 CSV 数据到内存结构体（运行时处理，不改变存储格式）
- 扫描代码/设计系统，使用检测规则匹配 AI Slop 特征
- 根据权重计算"AI Slop 分数"（0-100，越高越像 AI 生成）
- 生成检测报告

**检测方法**：

- 关键词匹配（字体名称、颜色值）
- 模式匹配（CSS 类名、布局结构）
- 统计特征（组件重复度、颜色使用分布）

**实现方式**：

- 使用 `encoding/csv` 包读取 CSV
- 与现有 `service.go` 中读取 CSV 的方式保持一致

#### 4.3 集成到审查工具

- `/audit` 工具：包含 AI Slop 检测部分
- `/critique` 工具：AI Slop 检测作为最重要的检查项

### 阶段五：添加细粒度优化命令

#### 5.1 性能优化工具（ui_ux_optimize）

**功能**：

- 检测性能问题（布局抖动、昂贵动画、未优化资源）
- 提供优化建议（使用 transform/opacity、懒加载、代码分割）

#### 5.2 颜色优化工具（ui_ux_colorize）

**功能**：

- 分析当前颜色使用
- 建议策略性颜色引入
- 避免 AI 颜色调色板

#### 5.3 动效优化工具（ui_ux_animate）

**功能**：

- 检测缺少动效的交互
- 建议有意义的动效
- 避免弹跳/弹性缓动

#### 5.4 文案优化工具（ui_ux_clarify）

**功能**：

- 检测不清晰的 UX 文案
- 改进按钮标签、错误消息、空状态文案

   // 在 BaseTools() 中添加
