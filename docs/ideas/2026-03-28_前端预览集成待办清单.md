# 2026-03-28 前端预览集成待办清单

## 背景

DigEino 侧能力已经具备：

- `write_preview_manifest`
- `apply_preview_patch`（支持 `manifest_path` / `artifact_id`）
- `export_preview_bundle`
- `History` 快照与 `revision` 乐观锁

当前待做项集中在宿主前端（DigFlow / DigPdf 控制台或自建前端）集成。

## 总体目标

- 前端可预览由 Agent 生成的 HTML/React 产物
- 用户可视化修改内容并转为结构化 `patches`
- 保存后刷新到新 revision
- 支持导出 zip

## P0（静态 HTML 预览）待办

- [ ] **预览容器**：实现 iframe 预览（`srcdoc` 或同域静态 URL）
- [ ] **加载 manifest**：按 `artifact_id` 或 `manifest_path` 拉取 `preview-manifest.json`
- [ ] **渲染入口**：读取 `entry` 对应 HTML 内容并展示
- [ ] **选择器策略**：统一使用 `data-uiux-id` 标记可编辑块
- [ ] **编辑能力 1**：文本编辑，输出 `html_text` patch
- [ ] **编辑能力 2**：图片地址/alt 编辑，输出 `html_attr` patch
- [ ] **保存按钮**：调用后端 `apply_preview_patch`，传 `base_revision`
- [ ] **冲突处理**：处理 `revision mismatch`（提示刷新并重试）
- [ ] **成功刷新**：保存成功后刷新到新 revision 的页面状态
- [ ] **导出**：调用 `export_preview_bundle` 并提供下载

## P1（React + content.json）待办

- [ ] **预览壳**：集成 Sandpack 或 Vite 沙箱
- [ ] **入口约定**：`kind=react_sandpack` 时读取入口与依赖文件
- [ ] **数据驱动约定**：文案/图片落到 `content.json`（或等价模型）
- [ ] **字段编辑面板**：编辑 JSON 字段并生成 `json_pointer` patch
- [ ] **热更新**：patch 成功后同步新内容到预览
- [ ] **错误提示**：JSON pointer 非法、路径不存在、类型不匹配时给用户反馈

## P2（增强，可选）待办

- [ ] **源码级修改面板**：受控 `literal_replace` 能力（高风险操作二次确认）
- [ ] **历史浏览**：读取 `preview/history/{artifact_id}/...` 并展示版本列表
- [ ] **一键回滚**：从 history 选版本后恢复目标文件（前端发起，后端执行）
- [ ] **审查串联**：保存后可选触发 `ui_ux_audit` / `ui_ux_critique`

## 前后端接口约定（前端需对齐）

- [ ] 请求统一字段：
  - `artifact_id`（推荐）或 `manifest_path`
  - `base_revision`
  - `patches[]`
- [ ] 响应统一字段：
  - `ok`
  - `revision`
  - `updated_files`
  - `message`
- [ ] 错误码映射：
  - `revision mismatch`
  - `selector matches no elements`
  - `file extension not allowed`
  - `artifact_id not found`

## 验收标准（前端）

- [ ] 用户能在页面上改一个标题并保存，刷新后仍正确
- [ ] 用户能改图片 `src` 并保存，刷新后仍正确
- [ ] 两人并发编辑时，后保存者收到 revision 冲突提示
- [ ] 能下载 zip，且包含 manifest + entry + assets + editable_model
- [ ] React 模式下能改 `content.json` 字段并实时反映
