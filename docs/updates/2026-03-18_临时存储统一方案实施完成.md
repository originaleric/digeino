## 新增内容

### 1. 配置

```yaml
UIUX:
  TempStorage:
    BaseDir: "storage/temp"   # 会话模式临时根目录
```

### 2. 程序化 API

```go
import "github.com/originaleric/digeino/pkg/tempstorage"

path, err := tempstorage.SaveForReview(ctx, "designer_output.css", content)
// ctx 需包含 workspace_path 或 agent_session_id
```

### 3. LLM 工具

- **write_review_file**：参数 `filename`、`content`，根据 context 写入并返回绝对路径
- 已通过 `tools.BaseTools()` 自动注册

### 4. Context 键

- `workspace_path`：工作空间模式（DigFlow 已使用）
- `agent_session_id`：会话模式（DigPdf 已使用）

## DigPdf 迁移

- 移除 `saveAgentTempFile` 函数
- 所有调用点改为 `tempstorage.SaveForReview(reviewCtx, filename, content)`
- 保留 `ExtensionKeyDesignerCssPath`、`ExtensionKeyHtmlOutputPath` 键名
- 路径兼容：cwd = 项目根时，`storage/temp/agent/{sessionID}` 与原有路径一致

## DigFlow 评估

**结论**：DigFlow 无需改动即可使用 `write_review_file`。

- DigFlow 已在执行时注入 `workspace_path`，`write_review_file` 会优先使用该路径
- 通过 `digeinoTools.BaseTools(ctx)` 加载时，`write_review_file` 已包含在内
- **可选优化**：在 UI Pro 的 agent.yml 中，可将审查流程的 `write_file` + `get_workspace_path` 替换为单一 `write_review_file` 调用，简化 Prompt 引导；保留现有方式亦可工作
