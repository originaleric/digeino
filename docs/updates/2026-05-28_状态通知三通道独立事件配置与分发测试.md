# 2026-05-28 状态通知三通道独立事件配置与分发测试

## 背景

DigFlow embedded `Invoke/Stream` 通过 `webhook.NewConfiguredCollector` 装配通知 sink。DigEino 需统一承载飞书/微信/企业微信运行时通知，并支持多通道独立事件过滤。

## 本次更新

### 1. 通用 `StatusNotifier` 与 Collector 改造

| 文件 | 说明 |
| --- | --- |
| `webhook/notifier.go` | 新增 `StatusNotifier` 接口 |
| `webhook/status_collector.go` | `notifiers []StatusNotifier` 替代仅 `feishuNotifiers`；`AddNotifier` / 保留 `AddFeishuNotifier` 兼容 |
| `webhook/status_message.go` | 统一状态文本 `formatStatusTextMessage` |
| `webhook/feishu_notifier.go` | 复用统一文本格式 |

### 2. 微信 / 企业微信 Notifier

| 文件 | 说明 |
| --- | --- |
| `webhook/wechat_notifier.go` | `WeChatNotifier`，复用 `tools/wx.SendWeChatTextMessage` |
| `webhook/wecom_notifier.go` | `WeComNotifier`，复用 `tools/wx.SendWeComMessage`，按 `Applications[].ToUser` 投递 |

### 3. 三通道独立 `NotifyOnEvents`

文件：`config/config.go`

- `WeChatConfig.NotifyOnEvents`
- `WeComConfig.NotifyOnEvents`
- 默认 `["failed", "completed"]`

文件：`webhook/bootstrap.go`

- 各 Notifier 分别读取 `Feishu` / `WeChat` / `WeCom` 的 `NotifyOnEvents`，不再共用飞书事件列表

### 4. 配置校验与可观测性

文件：`webhook/config.go`

- `GetWeChatConfig` / `GetWeComConfig` 校验启用条件
- 飞书已启用但 `AppID`/`AppSecret` 为空时，一次性日志：`runtime notification sink disabled`

文件：`config/config.go`

- `WeComApplication.ToUser` 注释：DigEino **运行时状态通知**专用，非企业微信原生 Tool 参数字段
