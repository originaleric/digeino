### 2026-02-16 (v1.0.4) 官方 Eino 搜索组件集成与 WebSearch 配置重构

本次更新通过引入官方 `eino-ext` 组件，显著增强了 `DigEino` 的搜索稳定性，并对 WebSearch 的配置结构进行了标准化重构。

- **官方组件适配**：集成 `googlesearch`, `bingsearch` 和 `duckduckgo` 官方扩展组件，替换原有手动 HTTP 实现。
- **混合动力架构**：
    - **官方驱动**：Google, Bing, DuckDuckGo。
    - **自定义驱动**：保留并优化了 Bocha (博查) 和 SerpApi 引擎。
- **配置结构重构**：由单一 Key/Url 平铺结构升级为**嵌套式引擎特定配置**，支持为不同引擎设置独立的 API Key、Cx、Region 及 SafeSearch。
- **默认引擎策略**：将 `bocha` 设为默认搜索引擎，并实现了配置缺失时的自动降级与兜底逻辑。
- **统一输出格式**：重构了 `web_search` 工具的归一化层，确保所有引擎返回的 `SearchResult`（Title/URL/Description）结构完全一致。

## 配置示例 (Updated WebSearch Configuration)

在 `config.yaml` 或 `eino.yml` 中，现在的配置结构如下：

```yaml
Tools:
  WebSearch:
    Engine: "bocha" # 选填: bocha, serpapi, google, bing, duckduckgo
    Bocha:
      ApiKey: "您的博查 Key"
    Google:
      ApiKey: "Google API Key"
      Cx: "Search Engine ID"
    Bing:
      ApiKey: "Bing API Key"
      MaxResults: 10
      Region: "zh-CN"
    DuckDuckGo:
      MaxResults: 10
      SafeSearch: "moderate"
```

## 变更面 (Impact)

- **核心包**：`github.com/originaleric/digeino/tools/research`
- **配置包**：`github.com/originaleric/digeino/config`
- **依赖增加**：引入了 `cloudwego/eino-ext` 下的多个 search components。

## 升级建议
由于配置结构从平铺变为嵌套，建议手动更新配置文件中的 `WebSearch` 节点以适配新版本。
