# 2026-03-07 企业微信客服消息多类型支持

## 概述

在「企业微信 → 个人微信」客服通道上，除原有文字消息外，新增**图片、语音、视频、文件、图文链接、小程序卡片**六种消息类型的发送能力。通过 6 个独立工具（`send_wecom_customer_image` 等）暴露，与现有「发给企业成员」的 `send_wecom_message` / `send_wecom_image` / `send_wecom_text_card` 设计一致，便于按场景选用。

## 新增工具

| 工具名 | msgtype | 主要参数 |
|--------|---------|----------|
| send_wecom_customer_image | image | open_kf_id, customer_id, media_id |
| send_wecom_customer_voice | voice | open_kf_id, customer_id, media_id |
| send_wecom_customer_video | video | open_kf_id, customer_id, media_id [, title, description] |
| send_wecom_customer_file | file | open_kf_id, customer_id, media_id |
| send_wecom_customer_link | link | open_kf_id, customer_id, title, desc, url, thumb_media_id |
| send_wecom_customer_miniprogram | miniprogram | open_kf_id, customer_id, title, appid, thumb_media_id [, pagepath] |

- 图片/语音/视频/文件：需先通过企业微信「上传临时素材」接口获取 `media_id`。
- 图文链接：`thumb_media_id` 为封面图，同样需上传临时素材获得。
- 小程序：`thumb_media_id` 为卡片封面图；可选 `pagepath` 指定点击后跳转的小程序页面。
- 所有工具均支持可选 `access_token`；未传时由 DigEino 使用配置中 `ManageAllKFSession: true` 的应用自动获取 token 并调用客服 API。

## 变更文件

### tools/wx/wx_types.go

- 新增 6 组请求/响应类型：`SendWeComCustomerImageRequest/Response`、`SendWeComCustomerVoiceRequest/Response`、`SendWeComCustomerVideoRequest/Response`、`SendWeComCustomerFileRequest/Response`、`SendWeComCustomerLinkRequest/Response`、`SendWeComCustomerMiniprogramRequest/Response`。

### tools/wx/wecom_message.go

- 新增 `getWeComCustomerAccessToken(ctx)`：使用 ManageAllKFSession 应用的 secret 调用企业微信 gettoken 接口，供无 access_token 时使用。
- 新增 6 个发送函数：`SendWeComCustomerImage`、`SendWeComCustomerVoice`、`SendWeComCustomerVideo`、`SendWeComCustomerFile`、`SendWeComCustomerLink`、`SendWeComCustomerMiniprogram`。内部统一构造 `msgtype` 对应 body 后调用既有 `sendWeComCustomerMessageAPI`。

### tools/wx/wecom_tool.go

- 新增 6 个工具构造函数：`NewWeComCustomerImageTool`、`NewWeComCustomerVoiceTool`、`NewWeComCustomerVideoTool`、`NewWeComCustomerFileTool`、`NewWeComCustomerLinkTool`、`NewWeComCustomerMiniprogramTool`。

### tools/tools.go

- 在 `BaseTools` 中于 `wecomCustomerTool` 之后注册上述 6 个新工具。

## 兼容性

- 现有 `send_wecom_customer_message`（文字）行为与配置不变。
- 无需修改 eino.yml 即可使用新工具；继续依赖 WeCom 配置及至少一个 `ManageAllKFSession: true` 的应用（与客服文字消息相同）。
