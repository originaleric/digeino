# 更新：企业微信接收消息工具

**日期**: 2026-03-12
**类别**: 功能 / 工具

## 概述

新增企业微信客服回调接收消息功能，支持接收个人微信用户发送给企业微信机器人应用的消息。企业微信会**实时**通过 POST 请求将加密的消息推送到配置的回调 URL，解密后可直接获取消息内容，无需主动轮询。

本更新新增一个 Eino 工具 `receive_wecom_customer_message`，支持两种模式：实时回调模式和主动拉取模式。

## 新功能

### 1. 接收消息工具 (`receive_wecom_customer_message`)

支持两种模式接收个人微信用户发送给企业微信机器人应用的消息：

#### 实时模式（推荐）
- 通过回调实时接收消息，消息即时到达
- 企业微信会实时推送加密的 XML 消息到配置的回调 URL
- 解密后直接获取消息内容，无需额外 API 调用
- 低延迟、高效、可靠（企业微信会自动重试失败的推送）

#### 拉取模式
- 主动拉取最近 3 天内的消息
- 用于补拉历史消息或主动查询场景
- 需要从回调事件中获取 token 才能拉取

### 2. 回调处理器 (`WeComCallbackHandler`)

提供完整的回调处理能力：

- **URL 验证**（GET 请求）：首次配置时验证回调 URL 的有效性
- **消息接收**（POST 请求）：实时接收并解密企业微信推送的消息
- **消息加解密**：实现企业微信标准的 AES-256-CBC 加解密算法
- **消息处理回调**：支持设置 `OnMessage` 回调函数处理接收到的消息

### 3. 消息拉取接口 (`SyncCustomerMessages`)

- 调用企业微信 `/cgi-bin/kf/sync_msg` 接口拉取消息
- 支持分页拉取（使用 cursor）
- 最多拉取最近 3 天内的消息
- 单次最多 1000 条

## 配置

在 `config/config.yaml` 的 `WeCom` 下新增回调配置：

```yaml
WeCom:
  Enabled: true
  CorpID: "ww1234567890abcdef"
  QYAPIHost: "https://qyapi.weixin.qq.com"
  TokenFilePath: "storage/app/wecom/access_token.json"
  
  # 新增：客服回调配置
  Callback:
    Enabled: true                    # 是否启用回调接收消息功能
    URL: "https://your-domain.com/api/v1/wecom/callback"  # 回调 URL（必须是公网 HTTPS）
    Token: "your_callback_token"     # 回调 Token（企业微信管理后台配置）
    EncodingAESKey: "your_encoding_aes_key_43_chars_base64"  # 回调消息加密密钥（43位Base64编码）
    Path: "/api/v1/wecom/callback"   # 回调路径（相对路径，用于路由注册）
  
  Applications:
    - AgentID: 1000001
      AgentSecret: "your_app_secret"
      ManageAllKFSession: true       # 必须为 true，用于拉取消息
```

## 工具参数说明

| 工具名 | 参数 | 说明 |
|--------|------|------|
| `receive_wecom_customer_message` | `mode` (可选) | `"realtime"`（实时回调，默认）或 `"pull"`（主动拉取） |
| | `open_kf_id` (可选) | 客服账号ID，不传则拉取所有 |
| | `external_userid` (可选) | 外部联系人ID，不传则拉取所有 |
| | `limit` (可选) | 拉取数量，默认1000（仅拉取模式有效） |
| | `cursor` (可选) | 游标，用于分页（仅拉取模式有效） |

## 使用指南

### 1. 前置条件

- 已注册企业微信并创建客服账号
- 在企业微信管理后台配置回调 URL、Token、EncodingAESKey
- 应用必须配置 `ManageAllKFSession: true`（用于拉取消息）
- 回调 URL 必须是公网可访问的 HTTPS 地址

### 2. 配置回调

在企业微信管理后台：
1. 进入「客户联系」→「配置」→「API」
2. 配置回调 URL（如：`https://your-domain.com/api/v1/wecom/callback`）
3. 获取 Token 和 EncodingAESKey
4. 将配置填入 `config.yaml`

### 3. 在外部系统中集成回调处理器

由于 DigEino 是插件，不强制要求内置 HTTP 服务器，提供回调处理器供外部系统集成：

#### 使用 Gin 框架示例

```go
import (
    "github.com/gin-gonic/gin"
    "github.com/originaleric/digeino/tools/wx"
)

func setupWeComCallback(r *gin.Engine) {
    callbackHandler := wx.NewWeComCallbackHandler()
    
    // 设置消息处理回调（业务逻辑）
    callbackHandler.OnMessage(func(msg wx.CustomerMessage) error {
        // 实时处理接收到的消息
        fmt.Printf("实时收到消息: %s\n", msg.Text.Content)
        // TODO: 业务处理（存储、路由、自动回复等）
        return nil
    })

    // URL 验证（首次配置时）
    r.GET("/api/v1/wecom/callback", callbackHandler.VerifyURL)
    
    // 实时消息接收（用户发送消息时，企业微信会实时推送）
    r.POST("/api/v1/wecom/callback", callbackHandler.HandleMessage)
}
```

#### 实时消息处理流程

1. 用户发送消息 → 企业微信实时推送加密 XML 到回调 URL
2. DigEino 回调处理器验证签名并解密消息
3. 如果解密后直接包含消息内容，立即触发 `OnMessage` 回调
4. 外部系统在回调中处理业务逻辑（实时响应）

### 4. 主动拉取消息（补拉历史消息）

```go
import "github.com/originaleric/digeino/tools/wx"

func pullHistoryMessages(ctx context.Context, token string) {
    req := wx.SyncMessageRequest{
        Token: token,  // 从回调事件中获取
        Limit: 100,
    }

    resp, err := wx.SyncCustomerMessages(ctx, req)
    if err != nil {
        log.Fatal(err)
    }

    for _, msg := range resp.MsgList {
        fmt.Printf("拉取到消息: %s\n", msg.Text.Content)
        // TODO: 业务处理
    }
    
    // 如果还有更多消息，使用 next_cursor 继续拉取
    if resp.HasMore == 1 {
        req.Cursor = resp.NextCursor
        // 继续拉取...
    }
}
```

## 支持的消息类型

接收到的消息支持多种类型：

- **文本消息** (`text`)：包含 `content` 和可选的 `menu_id`
- **图片消息** (`image`)：包含 `media_id`
- **语音消息** (`voice`)：包含 `media_id`
- **视频消息** (`video`)：包含 `media_id`
- **文件消息** (`file`)：包含 `media_id`
- **位置消息** (`location`)：包含经纬度、名称、地址
- **链接消息** (`link`)：包含标题、描述、URL、封面图
- **名片消息** (`business_card`)：包含用户ID
- **小程序消息** (`miniprogram`)：包含标题、appid、页面路径、封面图
- **菜单消息** (`msgmenu`)：包含菜单项列表
- **事件消息** (`event`)：包含事件类型

## 技术细节

### 消息加解密

企业微信使用 **AES-256-CBC** 加密：

1. **验证签名**：`SHA1(sort(Token, timestamp, nonce, Encrypt))` == `msg_signature`
2. **Base64 解码**：`EncodingAESKey` → 32字节 AES 密钥
3. **AES 解密**：使用 AES-256-CBC 模式，IV 取 AESKey 前16字节
4. **解析 XML**：得到明文消息内容

### 回调消息格式

**GET 请求（URL 验证）**：
- `msg_signature`: 签名
- `timestamp`: 时间戳
- `nonce`: 随机数
- `echostr`: 加密字符串（需解密后返回）

**POST 请求（实时消息推送）**：
- 企业微信发送加密的 XML 消息
- URL 查询参数：`msg_signature`、`timestamp`、`nonce`
- 解密后可能直接包含消息内容，或仅包含事件通知（需要调用拉取接口）

## 注意事项

1. **权限要求**：应用必须配置 `ManageAllKFSession: true`
2. **回调 URL**：必须是公网可访问的 HTTPS 地址（企业微信要求）
3. **实时性**：回调消息是实时推送的，需要在 5 秒内返回 HTTP 200，否则企业微信会重试（最多3次）
4. **消息时效**：
   - 实时回调：无时效限制（实时推送）
   - 主动拉取：只能拉取最近 3 天内的消息
5. **消息类型**：支持文本、图片、语音、视频、文件、位置、链接、名片、小程序等多种类型
6. **并发处理**：回调可能并发，需要做好并发控制和线程安全
7. **错误处理**：
   - 实时回调失败：企业微信会自动重试，但需要记录日志
   - 拉取消息失败：需要记录日志，避免丢失消息
8. **加解密性能**：消息加解密是同步操作，建议使用高效的加解密库，避免阻塞
9. **消息限制**：不支持读取通过发送消息接口发送的消息（只能读取客户发送的）
