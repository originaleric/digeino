# DigEino Updates - 2026-01-02

## Overview
Enhanced the `StatusCollector` and improved type compatibility to better support complex integrations like `DigFlow`.

## Changes

### Webhook Module
- **`StatusCollector` Enhancements**:
    - Added `GetExecutionID()`: Allows retrieving the associated Execution ID.
    - Added `GetStatusHistory()`: Returns a snapshot of the execution status history.
    - Added `GetTotalUsage()`: Returns the aggregated Token Usage for the session.
- **Interface Decoupling**:
    - Refactored `StatusStoreInterface`: Removed `CreateExecution` from the base interface in the `webhook` package. This allows for cleaner separation of concerns, as execution creation is typically handled at the controller/application level.

### Status Module
- **Compatibility Fix**:
    - Standardized `StatusStore` implementation to match the updated `webhook.StatusStoreInterface`.

### Tools Module
- **`BaseTools` Setup**:
    - Initialized `BaseTools` function in `tools.go` to provide a central registry for shared cross-project tools.

## Versioning
- Released as **v1.0.2**.
