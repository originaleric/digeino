## 使用指南

### 工具一：browser_snapshot（浏览器快照工具）

**功能**：获取页面结构化快照，提取可交互元素信息，返回元素引用ID（e0, e1, e2等）。

**适用场景**：

- 需要了解页面的可交互元素（按钮、链接、输入框等）
- 需要稳定的元素引用，避免 CSS 选择器失效
- 需要降低 Token 消耗（相比完整 HTML 降低 5-13 倍）

**参数说明**：

- `url` (必填)：目标网页 URL
- `filter` (可选)：过滤类型
  - `interactive`：仅可交互元素（默认）
  - `visible`：仅可见元素
  - `all`：全部元素
- `max_depth` (可选)：最大深度，-1 表示不限制（默认 -1）
- `wait_selector` (可选)：等待目标元素出现后再提取
- `use_cookie_domain` (可选)：Cookie 域名，用于加载并复用该域的 Cookie

**使用示例**：

```json
// 示例1：获取可交互元素快照
{
  "url": "https://example.com/login",
  "filter": "interactive"
}

// 示例2：等待特定元素后获取快照
{
  "url": "https://example.com",
  "filter": "interactive",
  "wait_selector": "#main-content"
}

// 示例3：使用 Cookie 复用登录态
{
  "url": "https://example.com/dashboard",
  "filter": "interactive",
  "use_cookie_domain": "example.com"
}
```

**返回示例**：

```json
{
  "url": "https://example.com/login",
  "title": "Login Page",
  "elements": [
    {
      "ref": "e0",
      "role": "textbox",
      "name": "Email",
      "value": "",
      "disabled": false,
      "focused": false,
      "depth": 2
    },
    {
      "ref": "e1",
      "role": "textbox",
      "name": "Password",
      "value": "",
      "disabled": false,
      "focused": false,
      "depth": 2
    },
    {
      "ref": "e2",
      "role": "button",
      "name": "Sign In",
      "disabled": false,
      "focused": false,
      "depth": 2
    }
  ]
}
```

**使用流程**：

1. 调用 `browser_snapshot` 获取页面元素快照
2. 从返回的 `elements` 中找到目标元素（通过 `ref`、`role`、`name` 等）
3. 使用 `browser_action` 工具，通过 `ref` 或 `selector` 操作元素

### 工具二：browser_action（浏览器操作工具）

**功能**：执行浏览器交互操作，支持点击、输入、填充、悬停、滚动、聚焦、按键等操作。

**适用场景**：

- 需要与页面元素交互（点击按钮、填写表单等）
- 需要模拟人类操作（随机延迟）
- 需要执行复杂的操作序列

**参数说明**：

- `url` (必填)：目标网页 URL
- `action` (必填)：操作类型
  - `click`：点击元素
  - `type`：输入文本（需要 `text` 参数）
  - `fill`：填充表单（清空后输入，需要 `text` 参数）
  - `hover`：鼠标悬停
  - `scroll`：滚动（可指定 `scroll_x`、`scroll_y`）
  - `focus`：聚焦元素
  - `press`：按键（需要 `key` 参数，如 Enter、Tab、Escape 等）
- `ref` (可选)：元素引用ID（如 e0, e1, e2），优先使用
- `selector` (可选)：CSS 选择器，当 `ref` 未提供时使用
- `text` (可选)：输入文本（`type`/`fill` 操作需要）
- `key` (可选)：按键名称（`press` 操作需要）
  - 支持：`enter`、`tab`、`escape`、`backspace`、`delete`
  - 支持：`arrowup`、`arrowdown`、`arrowleft`、`arrowright`
- `scroll_x`、`scroll_y` (可选)：滚动偏移（`scroll` 操作）
- `human_like` (可选)：是否模拟人类操作（随机延迟）
- `use_cookie_domain` (可选)：Cookie 域名

**使用示例**：

```json
// 示例1：点击按钮（使用 selector）
{
  "url": "https://example.com/login",
  "selector": "#submit-btn",
  "action": "click",
  "human_like": true
}

// 示例2：填写表单（使用 ref）
{
  "url": "https://example.com/login",
  "ref": "e0",
  "action": "fill",
  "text": "user@example.com"
}

// 示例3：输入文本
{
  "url": "https://example.com/search",
  "selector": "input[name='q']",
  "action": "type",
  "text": "search query",
  "human_like": true
}

// 示例4：按键操作（提交表单）
{
  "url": "https://example.com/search",
  "selector": "input[name='q']",
  "action": "press",
  "key": "enter"
}

// 示例5：滚动页面
{
  "url": "https://example.com/article",
  "action": "scroll",
  "scroll_y": 800
}

// 示例6：滚动到元素
{
  "url": "https://example.com/article",
  "selector": "#comments",
  "action": "scroll"
}
```

**返回示例**：

```json
{
  "success": true,
  "message": "操作 click 执行成功",
  "url": "https://example.com/dashboard"
}
```

**使用流程**：

1. 先调用 `browser_snapshot` 获取元素引用
2. 使用 `browser_action` 执行操作
3. 如需连续操作，可以复用同一个标签页（通过 `tab_id`）

### 工具三：browser_browse（增强版浏览器浏览工具）

**功能**：使用本地浏览器访问动态网页，支持多种提取模式和标签页复用。

**新增功能**：

- `mode` 参数：支持 `full`、`snapshot`、`summary` 三种模式
- `tab_id` 参数：支持标签页复用
- `content_selector` 参数：指定提取内容的范围

**参数说明**：

- `url` (必填)：目标网页 URL
- `action` (可选)：`read` 或 `screenshot`，默认 `read`
- `mode` (可选)：提取模式
  - `full`：完整内容提取（默认）
  - `snapshot`：快照模式（仅可交互元素，~800 tokens）
  - `summary`：摘要模式（页面摘要）
- `tab_id` (可选)：标签页ID，用于复用标签页
- `wait_selector` (可选)：等待目标元素出现后再提取
- `content_selector` (可选)：指定提取内容的范围（`mode=full` 时有效）
- `use_cookie_domain` (可选)：Cookie 域名

**使用示例**：

```json
// 示例1：完整内容提取（默认）
{
  "url": "https://example.com/article",
  "action": "read"
}

// 示例2：快照模式（Token 优化）
{
  "url": "https://example.com/article",
  "action": "read",
  "mode": "snapshot"
}

// 示例3：摘要模式
{
  "url": "https://example.com/article",
  "action": "read",
  "mode": "summary"
}

// 示例4：指定内容范围提取
{
  "url": "https://example.com/article",
  "action": "read",
  "mode": "full",
  "content_selector": ".article-content"
}

// 示例5：复用标签页
{
  "url": "https://example.com/page1",
  "action": "read",
  "tab_id": "tab_1234567890"
}

// 示例6：截图
{
  "url": "https://example.com",
  "action": "screenshot",
  "wait_selector": "#main-content"
}
```

**返回示例**：

```json
// mode=full 返回
{
  "url": "https://example.com/article",
  "title": "Article Title",
  "tab_id": "tab_1234567890",
  "text": "文章正文内容...",
  "markdown": "# 文章标题\n\n文章正文内容..."
}

// mode=snapshot 返回
{
  "url": "https://example.com/article",
  "title": "Article Title",
  "tab_id": "tab_1234567890",
  "elements": [
    {"ref": "e0", "role": "button", "name": "Share"},
    {"ref": "e1", "role": "link", "name": "Read More"}
  ]
}

// mode=summary 返回
{
  "url": "https://example.com/article",
  "title": "Article Title",
  "tab_id": "tab_1234567890",
  "summary": "标题: Article Title\n\n描述: 文章摘要...\n\n- 第一段内容\n- 第二段内容..."
}
```

### 最佳实践

1. **Token 优化**：
   - 优先使用 `browser_snapshot` 获取页面结构（~800 tokens）
   - 需要完整内容时使用 `browser_browse` 的 `full` 模式
   - 仅需摘要时使用 `summary` 模式

2. **元素定位**：
   - 优先使用 `ref`（元素引用），更稳定
   - `ref` 不可用时使用 `selector`（CSS 选择器）
   - 通过 `browser_snapshot` 获取 `ref` 后再操作

3. **标签页复用**：
   - 连续操作同一网站时，使用 `tab_id` 复用标签页
   - 减少浏览器启动开销，提升性能

4. **Cookie 管理**：
   - 使用 `use_cookie_domain` 复用登录态
   - Cookie 按域名存储，自动加载和保存

5. **人类模拟**：
   - 需要绕过反爬时，使用 `human_like: true`
   - 自动添加随机延迟，模拟人类操作

6. **错误处理**：
   - 操作失败时检查返回的 `success` 和 `message` 字段
   - 元素不存在时，重新获取快照更新 `ref`

## 注意事项

1. **元素引用（ref）的使用**：
   - `ref`（如 e0, e1, e2）仅在当前页面会话有效
   - 页面刷新或导航后，需要重新获取快照更新 `ref`
   - `ref` 比 CSS 选择器更稳定，但仍有页面变化的风险

2. **标签页复用**：
   - `tab_id` 由工具自动生成，在响应中返回
   - 标签页有 TTL（默认 30 分钟），过期后自动清理
   - 连续操作同一网站时，保存并使用返回的 `tab_id`

3. **Cookie 管理**：
   - Cookie 按域名存储在 `CookieStoreDir` 目录
   - 文件格式：`<domain>.json`
   - 首次访问时自动保存，后续访问自动加载

4. **性能考虑**：
   - 快照模式适合快速了解页面结构
   - 完整模式适合需要详细内容的场景
   - 摘要模式适合快速浏览和筛选

5. **错误处理**：
   - 操作失败时，检查返回的 `success` 和 `message` 字段
   - 元素不存在时，重新调用 `browser_snapshot` 获取最新快照
   - 超时错误可能需要增加配置中的超时时间
