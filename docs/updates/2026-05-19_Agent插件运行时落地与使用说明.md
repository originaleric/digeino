# 2026-05-19 Agent 插件运行时落地与使用说明

## 概述

按 [DigEino 通用 Agent 插件运行时方案](../ideas/2026-05-19_DigEino通用Agent插件运行时与本地采集网关方案.md)，在 **DigEino 仓库内**完成插件运行时与本地采集网关实现。宿主 Knowledge / DigFlow 对接留待各宿主项目。

演进策略：**Keep → Wrap → Expose**

- **Keep**：保留 `tools.BaseTools()` 进程内调用，不破坏现有 DigFlow/Knowledge 接入。
- **Wrap**：`gateway/executor` 包装 Legacy 工具，统一 `ToolManifest` / `ToolCall` / `ToolResult`。
- **Expose**：经 HTTP、WebSocket Collector、MCP、stdio 对外暴露。

## 能力总览

| 方案阶段 | 内容 | DigEino 仓库 |
|---------|------|-------------|
| 一 | 协议 + HTTP Gateway + `browser.browse` | ✅ |
| 二 | Collector WebSocket 反向连接 + `dev-host` | ✅ |
| 三 | Knowledge 宿主绑定、任务表、入库 | ⏭ 宿主项目 |
| 四 | DigFlow 远程工具 Adapter | ⏭ 宿主项目（可用 `gateway/client`） |
| 五 | MCP Server + stdio | ✅ |

### 1.1 通用协议（`gateway/protocol`）

### 1.2 首批 Wrap 工具

| 网关名 | Legacy |
|--------|--------|
| `browser.browse` | `research.BrowserBrowse` |

- 工具白名单、超时、`max_output_bytes`

### 1.4 HTTP API

| 方法 | 路径 | 说明 |
|------|------|------|
| GET | `/health` | 健康检查 |
| GET | `/manifest` | 工具清单 |
| POST | `/tools/call` | 执行工具 |

### 1.5 启动与 HTTP 调用示例

```bash
go run ./cmd/digeino gateway --config config/config.yaml
```

```bash
curl -s http://127.0.0.1:8787/manifest

curl -s -X POST http://127.0.0.1:8787/tools/call \
  -H 'Content-Type: application/json' \
  -H 'Authorization: Bearer your-token' \
  -d '{
    "type": "tool_call",
    "id": "call_wechat_001",
    "tool": "wechat.article.read",
    "input": {
      "url": "https://mp.weixin.qq.com/s/...",
      "format": ["text", "markdown"]
    },
    "policy": {
      "timeout_ms": 90000,
      "allowed_domains": ["mp.weixin.qq.com"]
    }
  }'
```

## 二、阶段二：Collector（WebSocket 反向连接）

### 2.1 能力

1. **WebSocket 反向连接**（`gateway/collector`）
   - Collector 主动连接 `wss://{host}/digeino/v1/collector/ws`
   - Bearer / `X-Digeino-Token` 鉴权，断线自动重连

2. **线协议**（`gateway/protocol/wire.go`）
   - `collector_hello` / `collector_hello_ack`
   - `collector_manifest`、`instance_status`（兼心跳）
   - `pull_tasks` / `pull_tasks_ack`
   - `tool_call` / `tool_result`
   - `ping` / `pong`

3. **任务执行**
   - **推送**：宿主经 WS 下发 `tool_call`
   - **拉取**：`PullIntervalSec > 0` 时周期性 `pull_tasks`
   - 并发上限 `MaxConcurrentCalls`（默认 1）
   - `rate_limit_key` 内存冷却

4. **开发参考宿主**（`gateway/devhost`，非生产 Knowledge）
   - `digeino dev-host`
   - `POST /dev/enqueue`、`GET /dev/collectors`

### 2.2 Collector 配置

```yaml
Collector:
  ServerURL: "http://127.0.0.1:8790"
  WSPath: "/digeino/v1/collector/ws"
  Token: "dev"
  InstanceID: "collector_local_001"
  HeartbeatIntervalSec: 30
  PullIntervalSec: 5    # 0 = 仅推送
  PullBatchSize: 1
  MaxConcurrentCalls: 1
```

### 2.3 本地联调

终端 1 — 参考宿主：

```bash
go run ./cmd/digeino dev-host --addr :8790 --token dev
```

终端 2 — Collector：

```bash
go run ./cmd/digeino collector \
  --config config/config.yaml \
  --server http://127.0.0.1:8790 \
  --token dev \
  --instance-id collector_local_001 \
  --pull-interval 5
```

投递任务（队列 + pull）：

```bash
curl -s -X POST http://127.0.0.1:8790/dev/enqueue \
  -H 'Authorization: Bearer dev' \
  -H 'Content-Type: application/json' \
  -d '{
    "instance_id": "collector_local_001",
    "mode": "queue",
    "call": {
      "type": "tool_call",
      "id": "call_test_1",
      "tool": "browser.browse",
      "input": { "url": "https://example.com" },
      "policy": { "allowed_domains": ["example.com"] }
    }
  }'
```

推送模式： `"mode": "push"`，宿主立即下发 `tool_call`。

### 2.4 线协议摘要（供宿主实现）

| 方向 | type | 说明 |
|------|------|------|
| C→H | `collector_hello` | 建连 |
| H→C | `collector_hello_ack` | 确认 |
| C→H | `collector_manifest` | 工具清单 |
| C→H | `instance_status` | 心跳 |
| C→H | `pull_tasks` | 拉取任务 |
| H→C | `pull_tasks_ack` | 返回 `calls[]` |
| H→C | `tool_call` | 下发执行 |
| C→H | `tool_result` | 回传结果 |

## 三、阶段三～五：MCP、stdio、工具扩展与 Artifact

### 3.1 CLI 命令

```bash
digeino gateway     # HTTP Tool Gateway（:8787）
digeino collector   # WebSocket 反向连接
digeino mcp         # MCP Server（stdio，IDE）
digeino stdio       # 行 JSON 网关协议
digeino dev-host    # 本地参考宿主
```

### 3.2 已 Wrap → Expose 的工具

| 网关名 | 说明 |
|--------|------|
| `browser.browse` | 本地浏览器抓取 |
| `browser.snapshot` | 页面可交互元素快照 |
| `browser.action` | 浏览器交互操作 |
| `wechat.article.read` | 公众号文章采集 |
| `file.read` | 本地读文件（需 `Gateway.AllowedReadPaths`） |

### 3.3 Artifact

- 目录：`storage/app/gateway_artifacts`
- `GET /artifacts/{id}` 下载
- `browser.browse` 截图自动入库

### 3.4 MCP

基于 `mark3labs/mcp-go`，从 `ToolManifest` 动态注册。Cursor 配置示例：

```json
{
  "mcpServers": {
    "digeino": {
      "command": "digeino",
      "args": ["mcp", "--config", "/path/to/config.yaml"]
    }
  }
}
```

### 3.5 stdio JSON 协议

每行一个 JSON：

```json
{"type":"get_manifest"}
```

```json
{"type":"tool_call","id":"c1","tool":"browser.browse","input":{"url":"https://example.com"}}
```

### 3.6 宿主 Go SDK

```go
import gwclient "github.com/originaleric/digeino/gateway/client"

c := gwclient.New("http://127.0.0.1:8787", "your-token")
manifest, _ := c.Manifest(ctx)
result, _ := c.Call(ctx, call)
data, _, _ := c.FetchArtifact(ctx, artifactID)
```

## 四、配置说明

`config/config.yaml`：

```yaml
Gateway:
  ListenAddr: ":8787"
  InstanceID: "digeino-local"
  AuthToken: ""
  AllowedTools:
    - browser.browse
    - browser.snapshot
    - browser.action
    - wechat.article.read
  AllowedReadPaths: []   # 非空时启用 file.read
  ArtifactEnabled: true
  ArtifactDir: "storage/app/gateway_artifacts"
  ArtifactTTLMinutes: 60

Collector:
  ServerURL: ""
  WSPath: "/digeino/v1/collector/ws"
  Token: ""
  InstanceID: "collector_local_001"
  PullIntervalSec: 0
  MaxConcurrentCalls: 1

Tools:
  LocalBrowser:
    Enabled: true
    AllowedDomains:
      - mp.weixin.qq.com
```

## 五、文档与 Wrap 开发

Gateway 文档位于代码包同级目录（由 `docs/gateway/` 迁入）：

| 文档 | 读者 | 内容 |
|------|------|------|
| [`gateway/README.md`](../../gateway/README.md) | 宿主实现方 | HTTP / WS / MCP / stdio、API、SDK |
| [`gateway/USAGE.md`](../../gateway/USAGE.md) | DigEino 开发者 | Wrap → Expose 步骤、模板、Checklist |

新增网关工具简要流程：

1. 在 `gateway/executor/` 实现 `XxxEntry()`
2. 在 `gateway/bootstrap.go` 的 `NewRegistry()` 中 `Register`
3. 更新 `Gateway.AllowedTools` / `Collector.AllowedTools`
4. `curl /manifest` 与 `POST /tools/call` 验证
