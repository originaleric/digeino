# 2026-05-09 飞书通知能力接入说明

## 一、本次更新概览

本次在 DigEino 中新增飞书通知能力，方案对齐 OpenClaw 主路径：

- 入站接入默认 `websocket`（可切换 `webhook`）
- 出站消息统一通过飞书开放平台 HTTP API 发送

同时支持两类触发方式：

1. 自动通知：状态事件触发发送（由配置 `NotifyOnEvents` 控制）
2. 手动通知：新增工具 `send_feishu_message`

---

## 二、核心改动

### 2.1 配置层新增 `Feishu` 模块

已在配置结构和默认配置中新增飞书相关 key：

- `Feishu.Enabled`
- `Feishu.ConnectionMode`
- `Feishu.EventIngest.*`
- `Feishu.SendViaAPI`
- `Feishu.NotifyOnEvents`
- `Feishu.API.*`

对应文件：

- `config/config.go`
- `config/config.yaml`

### 2.2 自动状态通知链路接入飞书

在状态采集器中新增飞书 sink，事件符合 `NotifyOnEvents` 时自动发送飞书消息。

对应文件：

- `webhook/feishu_client.go`（飞书 API 客户端、token 获取与缓存、消息发送与重试）
- `webhook/feishu_notifier.go`（状态消息格式化与事件过滤）
- `webhook/status_collector.go`（新增飞书 notifier 分发）
- `webhook/config.go`（新增飞书配置提取）
- `webhook/bootstrap.go`（默认装配飞书 notifier）

### 2.3 新增手动发送工具

新增工具：

- `send_feishu_message`

支持参数：

- `content`（必填）
- `receive_id_type`（可选）
- `receive_ids`（可选，不传则回退配置默认值）

对应文件：

- `tools/feishu/tool.go`
- `tools/tools.go`（工具注册）

### 2.4 文档补充

- 工具总览更新：`docs/tools_info.md`
- 飞书工具独立说明：`tools/feishu/README.md`

## 三、配置示例

```yaml
Feishu:
  Enabled: true
  ConnectionMode: "websocket"
  EventIngest:
    WebsocketEnabled: true
    WebhookEnabled: false
    VerificationToken: ""
    WebhookPath: "/feishu/events"
    WebhookHost: "127.0.0.1"
    WebhookPort: 3000
  SendViaAPI: true
  NotifyOnEvents: ["failed", "completed"]
  API:
    Enabled: true
    AppID: "cli_xxx"
    AppSecret: "xxx"
    BaseURL: "https://open.feishu.cn"
    Timeout: 5
    RetryCount: 2
    RetryDelayMs: 500
    ReceiveIDType: "chat_id"
    ReceiveIDs:
      - "oc_xxx"
```

## 四、兼容性与默认行为

- 默认 `Feishu.Enabled=false`，不会影响现有项目行为
- 不影响现有微信、企业微信与通用 webhook 通知能力
- 保持“配置开启即生效”的工具注册和通知装配模式
