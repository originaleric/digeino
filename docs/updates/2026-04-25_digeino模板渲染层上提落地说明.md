# 2026-04-25 digeino 模板渲染层上提落地说明

## 概述

本次在 `pkg/render` 内新增 `template` 子包，落地“业务 view model -> 安全 HTML 片段”的通用模板渲染能力：

- 业务项目只需提供模板文件与 view model；
- DigEino 统一承担模板加载、缓存、函数注入、路径约束与错误分类；
- 默认使用 Go 标准库 `html/template`，动态文本自动 HTML 转义；
- 不改变既有 `Parse -> []Block -> HTML/UIModel` 主链路，属于并行增量能力。

## 新增内容

### 1) `pkg/render/template` 子包

- `template.go`
  - 提供 `RenderTemplate(templateText string, data any, opts TemplateRenderOptions) (string, error)`；
  - 提供 `RenderTemplateFromFile(path string, data any, opts TemplateRenderOptions) (string, error)`；
  - 定义 `TemplateRenderOptions`，支持 `Funcs`、`Cache`、`CacheKey`、`BaseDir`、`Strict`；
  - 定义 `TemplateErrorKind` 与 `TemplateError`，便于业务侧用 `errors.As` 做降级处理。

### 2) 安全与缓存策略

- `html/template` 作为默认引擎，保持 HTML 转义默认开启。
- `BaseDir` 用于约束相对模板路径，拒绝 `../` 等目录逃逸。
- `Strict` 启用 `missingkey=error`，模板缺字段时返回执行错误。
- `Funcs` 做基础校验：函数名非空、符合模板函数命名规则、值必须为函数。
- 文件模板缓存按绝对路径、mtime、size、函数集与 strict 设置生成缓存键。
- inline 模板仅在 `Cache=true` 且设置 `CacheKey` 时启用缓存。

### 3) 文档更新

- `pkg/render/README.md`
  - 子包列表新增 `pkg/render/template`；
  - 新增「3a. 数据模板渲染（view model -> HTML 片段）」章节；
  - 说明模板渲染与 `pkg/render/html` 的边界：前者面向业务 view model，后者面向 `[]render.Block`；
  - 补充 `RenderTemplateFromFile` 调用示例与错误分类说明。

## 错误分类

新增错误类型：

- `not_found`：模板文件不存在；
- `path`：模板路径非法、目录逃逸或指向目录；
- `parse`：模板语法解析失败；
- `execute`：模板执行失败，例如 strict 模式下缺字段；
- `func`：函数注册非法。

业务侧可通过 `errors.As(err, &te)` 获取 `*rendertemplate.TemplateError`，再根据 `te.Kind` 做回退或降级。

## 接入建议

推荐业务项目保留：

- `config/templates/*.tmpl`：模板文件；
- `config/render.yaml` 或独立 CSS：样式策略；
- service 层 DTO -> view model 的数据准备逻辑。

示例：

```go
html, err := rendertemplate.RenderTemplateFromFile(
    "strategist_card.tmpl",
    viewModel,
    rendertemplate.TemplateRenderOptions{
        BaseDir: "config/templates",
        Cache:   true,
        Strict:  true,
        Funcs: map[string]any{
            "percent": formatPercent,
        },
    },
)
```

灰度期建议业务项目保留原本地渲染开关，确认输出一致后再删除重复模板加载、缓存和字符串拼装逻辑。
